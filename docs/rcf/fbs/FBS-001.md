## 3. Technical Specifications

### 3.1 Architecture Pattern
```
skills/
├── [skill-name]/
│   ├── SKILL.md           # Metadata and documentation
│   ├── index.js/py        # Main execution logic
│   ├── schema.json        # Input/output schema
│   ├── fixtures/          # Test fixtures
│   │   ├── input.json
│   │   └── expected.json
│   └── README.md          # Usage documentation
```

### 3.2 Technology Stack
- **Languages**: JavaScript (Node.js) or Python
- **Schema**: JSON Schema for validation
- **Output**: Thinking Card format (Markdown)
- **Testing**: Fixture-based validation

### 3.3 Data Flow
1. Skill receives input parameters
2. Validates against schema
3. Processes analysis logic
4. Formats output as Thinking Card
5. Returns structured response
